## 三、错误处理规范 / Error Handling

### 3.1 thiserror 定义

```rust
// src/error.rs

use thiserror::Error;

#[derive(Error, Debug)]
pub enum LshError {
    #[error("Element not found: {0}")]
    ElementNotFound(String),

    #[error("Invalid element ID: {0}")]
    InvalidElementId(String),

    #[error("Category conflict: {0} vs {1}")]
    CategoryConflict(String, String),

    #[error("Serialization error: {0}")]
    SerializationError(#[from] serde_json::Error),

    #[error("IO error: {0}")]
    IoError(#[from] std::io::Error),

    #[error("Lock error: {0}")]
    LockError(String),
}

pub type Result<T> = std::result::Result<T, LshError>;
```

### 3.2 错误传播

```rust
// 使用 ? 操作符传播错误
pub fn load_element(id: &str) -> Result<Element> {
    let data = read_file(id)?;          // IoError 传播
    let elem: Element = serde_json::from_slice(&data)?;  // SerializationError 传播
    Ok(elem)
}

// 避免 unwrap()/expect()
fn bad_example() -> Result<Element> {
    let elem = elements.get(id).unwrap();  // 禁止：panic 风险
    Ok(elem)
}

fn good_example() -> Result<Element> {
    let elem = elements.get(id)
        .ok_or_else(|| LshError::ElementNotFound(id.to_string()))?;
    Ok(elem.clone())
}
```

## 四、并发安全规范 / Concurrency Safety

### 4.1 锁使用规范

```rust
use parking_lot::RwLock;
use std::sync::Arc;

pub struct ElementManager {
    elements: RwLock<HashMap<ElementId, Element>>,
    version: AtomicU64,
}

impl ElementManager {
    // 读锁：多读单写
    pub fn get(&self, id: &ElementId) -> Option<Element> {
        let elements = self.elements.read();
        elements.get(id).cloned()
    }

    // 写锁
    pub fn insert(&self, elem: Element) -> Result<()> {
        let mut elements = self.elements.write();
        elements.insert(elem.id.clone(), elem);
        self.version.fetch_add(1, Ordering::SeqCst);
        Ok(())
    }

    // 批量操作
    pub fn batch_get(&self, ids: &[ElementId]) -> Vec<Option<Element>> {
        let elements = self.elements.read();
        ids.iter().map(|id| elements.get(id).cloned()).collect()
    }
}
```

### 4.2 Arc/Rc 选择

```rust
// 单线程：Rc<T>

    let data_clone = Arc::clone(&data);
    std::thread::spawn(move || {
        // ...
    });

// 需要写锁：Arc<RwLock<T>>

    std::thread::spawn(move || {
        let mut write = data_clone.write();

}
```

## 八、验证规则 / Validation Rules

### 8.1 LSH ID 统一格式

**唯一抽象源**：[lsh_format.md §2.2](../../lsh-protocol/docs/lsh_format.md#L56-L95)

**格式**：`{prefix}[_group...]-{timestamp}-{hash}`

| 组件 | 分隔符 | 说明 | 示例 |
|------|--------|------|------|
| prefix | - | 主类型，必须 | `device`, `theory`, `tnews` |
| group | `_` | 可选子类型，N个 | `classifier`, `gradient_paradigm` |
| timestamp | `-` | 创建时间 | `20260511` 或 `1778489606` |
| hash | - | 唯一指纹 | `a3f8c2`, `19970` |

**核心原则**：`-` 是唯一抽象，分隔语义部分与时间戳/哈希。

### 8.2 LSH ID 生成函数

```rust
/// 生成 LSH ID
/// 规范格式: {prefix}_{group}-{timestamp}-{hash}
/// 示例: tnews_classifier-1778489606-19970
fn generate_lsh_id(prefix: &str) -> String {
    use std::time::{SystemTime, UNIX_EPOCH};
    let timestamp = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_secs();
    let hash = format!("{:x}", timestamp % 0xFFFFFF);
    format!("{}-{}-{}", prefix, timestamp, hash)
}
```

**兼容格式**：下划线分隔（`prefix_timestamp_hash`）仍支持，但不推荐用于新项目。
