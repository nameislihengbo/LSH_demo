## 二、链式思考 (Chain of Thought)

### 2.1 CoT 模板

```markdown
## 思考链

当你遇到 [问题类型] 时，请执行以下思考链：

### 步骤 1：问题解构
- 输入是什么？
- 期望输出是什么？
- 关键约束条件有哪些？

### 步骤 2：方案搜索
- 相关知识有哪些？
- 类似问题如何解决？
- 可行的方案有几个？

### 步骤 3：方案评估
- 各方案的优缺点？
- 风险和成本？
- 最优方案是？

### 步骤 4：执行计划
- 具体步骤？
- 需要什么资源？
- 预期结果？

### 步骤 5：验证迭代
- 结果是否正确？
- 能否优化？
- 学到了什么？
```

### 2.2 CoT 触发条件

```markdown
## CoT 触发条件

当满足以下任一条件时，自动启用思考链：
1. 用户问题包含"为什么"、"如何"、"分析"
2. 问题涉及多个变量或关系
3. 需要推理或计算
4. 答案不明确或有多解

[示例]
用户: "为什么 LSH-SFA 的梯度会爆炸但模型还能收敛？"
触发: [分析推理 L2] → [执行思考链]
```

```markdown
## 示例选择规则

1. **相似性优先**：选择与当前问题最相似的示例
2. **多样性覆盖**：确保示例覆盖不同情况
3. **难度递进**：从简单到复杂排序
4. **数量控制**：通常 2-5 个示例

[选择算法]
```
1. 计算当前问题与各示例的相似度
2. 按相似度排序

4. 确保覆盖边界情况

## 四、记忆系统

### 4.1 记忆分层

```markdown
## 记忆系统

### 短期记忆（当前会话）
- 用户当前问题
- 最近的交互内容
- 待处理的任务

### 中期记忆（会话间）
- 用户偏好设置
- 项目上下文
- 常用模式

### 长期记忆（持久化）
- 专业领域知识
- 成功案例库
- 失败教训库
```

### 4.2 记忆更新规则

```markdown
## 记忆更新

### 更新触发条件
1. 用户明确提供偏好
2. 相同错误发生 2 次以上
3. 任务成功完成

### 更新操作
```
记忆_更新(类型, 键, 值):
  如果 记忆不存在:
    创建新记忆
  否则:
    合并/覆盖
  标记: 需要持久化
```

## 六、提示词组合引擎

### 6.1 组合规则

```python
class PromptComposer:
    """提示词组合引擎"""

    def __init__(self):
        self.rules_loader = RulesLoader("rules/")

    def compose(
        self,
        role: str,
        task: str,
        context: dict
    ) -> str:
        # 1. 加载角色定义
        role_def = self.rules_loader.load(f"role_{role}")

        # 2. 加载任务规则
        task_rules = self.rules_loader.load(f"task_{task}")

        # 3. 加载上下文规则
        context_rules = self.rules_loader.load_context(context)

        # 4. 组合
        prompt = self._merge(
            role_def,
            task_rules,
            context_rules
        )

        # 5. 验证
        self._validate(prompt)

        return prompt
```

## 七、超级规则增强

### 7.1 规则增强机制

```markdown
## 超级规则增强

### 自动规则应用
根据 [输入特征]，自动应用相关规则：

| 输入特征 | 触发规则 | 增强操作 |
|----------|----------|----------|
| 包含代码 | code_foundation | 添加代码规范检查 |
| 包含数学 | math_notation | 应用数学符号规则 |
| 涉及论文 | paper_foundation | 添加论文格式要求 |
| 涉及项目 | project_foundation | 添加架构规范 |

### 规则冲突解决
当多条规则冲突时：
1. 优先级高的规则优先
2. 同优先级：更具体的规则优先
3. 无法解决：请求用户澄清
```
