# 苏州昆山人工智能单人创业（OPC）大赛参赛计划书

## 一、项目概述

### 1.1 项目名称

**万象天工 - 通用具身智能基座**

### 1.2 项目定位

本项目是一个**通用具身智能基座平台**，构建"感知-认知-决策-执行"的完整AI能力闭环。

**当前阶段**：已完成设备控制、家居管理、多模态对话等MVP核心功能。

**核心能力**：

- **多模态智能交互**（已初步实现）：语言、视觉、语音的统一理解与生成
- **物理世界执行**（已初步实现）：机器人控制、设备操作、任务执行
- **虚拟世界构建**（规划中）：文生世界，从文本/图像自动生成3D场景

**首发应用**：万象天工智能管家，面向个人与家庭的AI全能管家。

### 3.1 系统架构（已实现）

```
┌─────────────────────────────────────────────────────────┐
│                   前端层（PySide6）                       │
│     主界面 │ 对话窗口 │ 设备管理 │ 机器人控制             │

│     事件总线 │ 信号绑定 │ 状态管理                        │

│                   API通信层                              │
│     HTTP Client │ WebSocket │ REST API                  │
├─────────────────────────────────────────────────────────┤

│     用户服务 │ 设备服务 │ 对话服务 │ 机器人服务           │
├─────────────────────────────────────────────────────────┤
│                   AI模型层                               │
│     Qwen3-VL │ Whisper │ Edge-TTS │ ChromaDB            │
```

#### 3.2.4 LSH 视图同步协议

**Li Sheng Home View Synchronization Protocol** - 自研的空间数据视图同步协议，专为智能家居 3D 可视化场景设计。

| 特性 | 说明 |
|------|------|
| 核心思想 | "位置优先"——所有事件携带位置信息 |
| 架构模式 | 发布-订阅模式，视图完全解耦 |
| 坐标系统 | 统一右手坐标系，2D/3D 视图自动适配 |

**已实现功能**：

| 功能 | 技术方案 | 状态 |
|------|---------|------|
| 事件系统 | 20+ 事件类型（房间/设备/组件/物品） | ✅ 已实现 |
| 批量模式 | 事件合并，避免重复刷新 | ✅ 已实现 |
| 坐标适配 | 2D 翻转 Y 轴，3D 无需适配 | ✅ 已实现 |
| 视图同步 | 结构视图/2D 视图/3D 视图/列表视图 | ✅ 已实现 |

**协议优势**：

| 对比项 | LSH 协议 | 传统方案 |
|--------|---------|---------|
| 耦合度 | 低（发布-订阅） | 高（直接调用） |
| 扩展性 | 新增视图只需订阅 | 需修改所有数据操作 |
| 空间语义 | 原生支持位置数据 | 无 |
| 批量优化 | 内置事件合并 | 需手动实现 |
